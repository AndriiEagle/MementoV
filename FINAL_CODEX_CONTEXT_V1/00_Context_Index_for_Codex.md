## 2. Что здесь зафиксировано окончательно

### Frozen product truth

- Продукт = `CHROMA + ECHO + Conversational Biographer + Memory Router`
- `ECHO` — первый functional MVP
- `CHROMA` — обязательный emotional and brand pillar
- Night-first UX — центральный сценарий
- MVP без обязательной регистрации
- Email только после ценности
- Family Buyer — главный ранний коммерческий вход
- Elderly User — главный activation/retention user
- UI-языки: `DE-CH`, `EN`, `FR-CH`, `IT-CH`, `RM`
- `Swiss German / Standard German` — conversational layer AI
- Продукт не является therapist substitute
- Продукт не обещает literal immortality
- Продукт не должен строить deadbots, post-mortem avatars или synthetic “messages from the dead”
- Privacy by design и Swiss trust — не маркетинг, а архитектурный слой
- Sensitive voice/legacy data должны проектироваться с приоритетом Swiss/EU trust model
- User data не должны использоваться для foundation model training
- DOM-first narrative architecture обязательна
- Full VN engine, heavy 3D и scope explosion запрещены в первой волне

## 5. Как Codex должен использовать этот контекст

- Не перепридумывать концепцию с нуля.
- Не спорить с frozen truths без новых сильных оснований.
- Не раздувать scope ради “ещё одной классной идеи”.
- Сначала следовать `Final_Codex_Swarm_Development_Bible_v1.md`.
- Для параллельной работы следовать `Swarm_Task_Packets_v1.md`.
- Для проверки спорных точек использовать `Deep_Research_Prompts_Critical_Points_v1.md`.

## 6. Что дальше не нужно делать

- Не создавать новые концептуальные документы без серьёзной причины.
- Не смешивать MVP и Phase 2.
- Не добавлять auth, social graph, heavy 3D, full AGI worldbuilding в первую волну.
- Не оптимизировать под vanity metrics вроде session length ради зависимости.

## 7. Что дальше нужно делать

- Либо запускать research по критическим точкам;
- либо начинать работу роя агентов по пакетам;
- либо готовить production assets: copy, prompts, localizations, legal, motion/audio spec.
